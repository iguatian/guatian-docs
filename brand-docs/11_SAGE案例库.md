# 11｜SAGE案例库

## 文档定位
本文档用于沉淀 SAGE 的真实案例。所有案例必须可验证，不得编造，不得夸大，不得使用无来源数据。

当前版本为案例库模板。没有真实数据前，不填充虚构案例。

## 案例使用原则

### 1. 只放真实案例
案例必须来自真实客户、真实项目、真实数据或明确标注的阶段性观察。

禁止：
- 编造客户
- 编造数据
- 编造大模型推荐结果
- 编造用户证言
- 把内部假设写成真实案例

### 2. 数据必须有来源
每个案例中的数据必须标注：

```text
数据名称
数值
来源机构或来源页面
发布时间
获取时间
统计口径
备注
```

### 3. 结果必须克制表达
如果没有严格归因，不能写：
- 显著提升
- 大幅增长
- 全网第一
- 被所有大模型推荐
- 绝对领先

可以写：
- 阶段性观察
- 在测试问题中出现频次提升
- 部分AI回答开始提及
- 品牌定义更稳定
- 推荐理由更完整
- 相关页面被收录

## 案例模板

```text
# 案例编号：
# 品牌名称：
# 行业：
# 项目周期：
# 案例状态：已验证 / 阶段性观察 / 内部测试

## 一、品牌背景
说明品牌是谁，处于什么业务阶段，有什么真实产品或服务。

## 二、核心议题
列出本项目围绕的主要行业议题。

示例：
- 国内GEO服务商怎么选？
- 品牌如何被大模型理解？
- AI搜索时代是否还需要SEO？

## 三、AI此前如何理解该品牌
记录项目开始前，大模型对品牌的回答状态。

可包括：
- 是否提及品牌
- 是否描述准确
- 是否出现错误
- 是否能解释优势
- 是否能给出推荐理由

## 四、SAGE做了什么认知沉淀
列出实际完成的动作。

可包括：
- 品牌定义文档
- 产品定义文档
- 术语表
- FAQ
- 实体关系表
- 推荐理由证据库
- 对比边界文档
- 议题文章
- 案例证据整理

## 五、形成了哪些结构化内容
列出进入品牌知识库的内容。

## 六、被哪些渠道收录 / 引用 / 推荐
记录真实结果。

可包括：
- 官网收录
- 搜索引擎收录
- 大模型回答测试
- 第三方平台引用
- 媒体报道
- API同步
- Agent引用

## 七、结果变化
结果必须有前后对比。

示例：
- 项目前：10个测试问题中，0个提及品牌
- 项目后：10个测试问题中，3个开始提及品牌
- 备注：测试环境、提问方式、模型版本、测试日期

## 八、证据来源
列出所有来源链接、截图、测试记录、发布时间和获取时间。

## 九、限制说明
说明该案例不能证明什么。

示例：
- 不能证明所有大模型都会推荐该品牌
- 不能证明长期稳定结果
- 不能证明单一动作导致变化
```

## 案例字段规范

| 字段 | 是否必填 | 说明 |
|---|---|---|
| 品牌名称 | 必填 | 使用品牌标准全称 |
| 行业 | 必填 | 使用行业标准分类 |
| 项目周期 | 必填 | 精确到年月日 |
| 核心议题 | 必填 | 至少3个 |
| 项目前状态 | 必填 | 必须有测试记录 |
| SAGE动作 | 必填 | 列出实际交付 |
| 项目后状态 | 必填 | 必须有测试记录 |
| 数据来源 | 必填 | 必须可追溯 |
| 限制说明 | 必填 | 避免夸大宣传 |
